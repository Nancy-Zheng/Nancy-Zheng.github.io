如何在win10中调出Windows照片查看器

win10自带的“画图”

win7自带的“Windows照片查看器”

说明

先给大家明确一点：其实“Windows照片查看器”还在Win10中的，不过被隐藏了罢了。如果你是从Win7升级上来的，那么这个应用还在。如果你买的电脑本来就是Win10系统，这个应用是看不到的。下面教大家找回来！

搜索“注册表编辑器”,依次打开以下目录

HKEY_LOCAL_MACHINE —> SOFTWARE —> Microsoft —> Windows Photo Viewer —> Capabilites —> FileAssociations
计算机\HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows Photo Viewer\Capabilities\FileAssociations

  • first: HKEY_LOCAL_MACHINE
  • next: SOFTWARE
  • next: Microsoft
  • next: Windows Photo Viewer
  • next: Capabilites
  • next: FileAssociations

右击，新建，字符串值

双击新建的数值，“编辑字符串”，填入字符“PhotoViewer.FileAssoc.Tiff”

按照上述方式添加其他图片格式

打开任意一张图片，选择“Windows照片查看器”进行查看

Vue+iView 实现主题切换（换肤）

Less

iView

实现

iView定制主题

View UI（iVIew）默认提供了一套 UI 主题，并且可以在一定程度上定制新主题，以满足业务和品牌上的多样化视觉需求。
View UI（iVIew）的样式是基于 Less 进行开发的，默认以前缀 .ivu- 作为命名空间，并且定义了一套样式变量，定制主题，就是编辑这个变量列表。

根据项目需要覆盖iView原先的样式

  • 在项目src目录下新建一个styles文件夹 ,包含iview文件夹（”src/styles/iview”）和my-theme文件夹（”src/styles/my-theme”）
  • 从node_modules下复制iview的样式文件夹”node_modules/iview/src/styles”
  • 将复制的文件夹拷贝至”src/styles/iview”下


  • 在main.js下引入你自己的样式src/styles/my-theme/index.less
  • 在每个主题下引入iview的样式文件，然后去覆盖

favicon.ico

浏览器无法正确解析 favicon.ico

今天碰到一个很奇怪的现象，同样的代码，打包之后上传服务器之后，不同 URL 在浏览器下渲染 favicon.ico 文件的效果不同，其中一个渲染失败，后来经过查找资料发现有网友的解决方法是用了 “/favicon.ico”

  • 项目代码

文件目录如下：

  • 解决办法

猜测原因：
假设正常情况下，百度的.cio 的 URL 是：https://www.baidu.com/favicon.ico

<%= BASE_URL %> 可能在域名结尾的时候没有匹配上”/“,导致 URL 解析为”https://www.baidu.comfavicon.ico"

favicon.ico 图标

favicon.ico 图标是网站的缩略标志，可以显示在浏览器标签、地址栏左边和收藏夹，是展示网站个性的缩略 logo 标志，也可以说是网站头像，如果要让网站看起来更专业、更美、更有个性，favicon.ico 是必不可少的，所以你在本站使用 ICO 图标转换工具就可以轻松达到你的要求 。

从特定的技术角度看，favicon 也并不只是仅仅让网站给人更专业的观感，也可以在一定程度上减轻服务器的流量带宽占用，一般为了提高网站的可用性，我们都会为自己的网站创建一个自定义的 404 错误文件，在这种情况下，如果网站没有相应的 favicon.ico 文件，每当有用户收藏网站/网页时， Web 服务器都会调用这个自定义的 404 文件，并在网站的错误日志中记录。这显然是应该予以避免的。

WPA

PWA（Progressive web apps，渐进式 Web 应用）运用现代的 Web API 以及传统的渐进式增强策略来创建跨平台 Web 应用程序。这些应用无处不在、功能丰富，使其具有与原生应用相同的用户体验优势。

Add to Home Screen
